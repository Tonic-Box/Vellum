# Terminal

[Back to index](index.md)

The terminal driver is internal. `App` talks only to the `Terminal` interface; the default
implementation is JLine-backed. Most applications never touch this package.

## Terminals

`com.tonic.vellum.terminal.Terminals` is the factory.

| Method | Description |
|---|---|
| `static Terminal system()` | Open the system terminal (JLine-backed). |

`App` calls this itself unless a terminal is otherwise provided.

## Terminal

`com.tonic.vellum.terminal.Terminal` is the swap-out seam. Any driver that satisfies it
(raw mode, alternate screen, normalized key input, a writable sink, clean restore) can
replace JLine without changes elsewhere. It extends `AutoCloseable`; `close()` calls
`restore()`.

| Method | Description |
|---|---|
| `void enterRawMode()` | Enter raw, no-echo mode. |
| `void enterAlternateScreen()` | Switch to the alternate screen buffer. |
| `void restore()` | Restore main screen, cooked mode, cursor shown. Idempotent. |
| `TerminalSize size()` | Current size in cells. |
| `void setResizeListener(Runnable)` | Callback on resize (may be off the UI thread). |
| `KeyEvent readKey(long timeoutMillis)` | Read and decode the next key; null on timeout. |
| `void write(String text)` | Queue output. |
| `void flush()` | Flush queued output. |
| `void hideCursor()`, `void showCursor()` | Cursor visibility. |
| `void moveCursor(int x, int y)` | Position the cursor at a zero-based cell. |

## TerminalSize

`com.tonic.vellum.terminal.TerminalSize` holds dimensions in cells.

| Method | Description |
|---|---|
| `int columns()` | Width in cells. |
| `int rows()` | Height in cells. |

## Notes

- Output is standard ANSI generated by the framework, not by the driver; a replacement
  driver only needs to provide the byte sink, key source, size, and mode control.
- Key decoding normalizes raw escape sequences into [KeyEvent](focus-and-input.md). Where a
  terminal cannot distinguish a chord such as Ctrl-Tab, that limitation is documented rather
  than faked.
- `Terminals.system()` fails fast with `IllegalStateException` when no interactive terminal
  is detected (a dumb terminal or non-TTY launcher), rather than emitting garbage.
